# wechat-claude-code_agent_bridge-skill 快速启动

## 1. 前置条件

- 已安装 Node.js 18+
- 已安装 `claude` CLI（Claude Code），并已登录或设置了 `ANTHROPIC_API_KEY`

  ```bash
  claude --version      # 确认可用（建议 2.1.x+）
  claude auth login     # 首次登录，或用 claude setup-token
  ```

- 已完成微信 ClawBot / ilink 绑定（下一步 `npm run setup` 会扫码）

## 2. 准备配置

```bash
cd wechat-claude-code_agent_bridge-skill/templates
cp bridge.config.example.json bridge.config.json
```

编辑 `bridge.config.json`，至少设置：

- `cwd`: Claude Code 执行任务的项目根目录（绝对路径）
- `claudeCodePath`: `claude` 可执行文件路径，默认 `claude`
- `permissionMode`: `default` / `acceptEdits`（默认推荐） / `plan` / `auto` / `dontAsk` / `bypassPermissions`

可选字段（进一步收紧权限边界）：

- `allowedTools`: 例 `["Bash(git:*)", "Edit", "Read"]` → 透传 `--allowedTools`
- `disallowedTools`: 例 `["Bash(rm:*)"]` → 透传 `--disallowedTools`
- `dangerouslySkipPermissions: true` → 传 `--dangerously-skip-permissions`，**仅在隔离环境使用**
- `bareMode: false`（默认）→ 桥接会继承你本机已经配好的全局 MCP server（`claude mcp add` 加过的）、`~/.claude/CLAUDE.md`、OAuth 等。如果你希望跑在最小隔离环境下（CI / 共享机器），改为 `true` 加 `--bare`
- `model`: 例 `sonnet` / `opus` / `claude-sonnet-4-6`

## 3. 安装依赖

```bash
npm install
```

## 4. 微信扫码登录

```bash
npm run setup
```

执行后会在 `templates/` 下生成 `credentials.json`。

## 5. 启动桥接

```bash
npm start        # 前台运行
npm run hot      # 热重载开发模式（src 文件变更自动重启）
```

后台运行推荐 tmux：

```bash
tmux new-session -d -s wechat-claude 'npm start'
tmux attach -t wechat-claude   # Ctrl+B D 退出
```

## 6. 微信里常用命令

```text
/help
/status
/model
/model sonnet
/model clear
/permission
/permission acceptEdits
/permission plan
/stop
/stopall
/clear
/send /tmp/result.txt
```

## 7. 说明

- **图片**：Claude Code CLI 没有 `--image` 参数。桥接会把图片下载解密后保存到 `.media_cache/inbound/`，然后把**路径**写进 prompt，让 Claude 用 `Read` 工具打开。因此 `allowedTools` 如果不含 `Read`，图片功能会失效。
- **文件 / 语音 / 视频**：落盘后把路径告诉 Claude 去读取。
- **追问**：忙时发送的消息会写入工作区根目录 `.bridge-followup.md`，Claude 可以在长任务中主动读取参考。
- **会话续接**：使用 `--resume <session-id>`，session id 从第一条 `system/init` 事件或 `result` 事件里抓。`sessionTimeoutMs`（默认 30min）内自动续接。
- **逐条审批**：本桥接**不支持**微信侧逐条批准工具调用（使用 `-p` 非交互模式）。如需精细控制，用 `permissionMode` + `allowedTools` + `disallowedTools` 三板斧。

## 8. 排障

- 启动后没有回复：先看 `tmux attach -t wechat-claude` 里的日志
- `claude: command not found`：把 `claudeCodePath` 改成 `which claude` 得到的绝对路径
- `--resume` 找不到 session：最近一次任务可能带了 `--no-session-persistence`，换个新消息重开即可（桥接不会主动传这个 flag）
- 权限被拒但希望放行：把对应工具加到 `allowedTools`，或临时 `/permission bypassPermissions`（谨慎）
